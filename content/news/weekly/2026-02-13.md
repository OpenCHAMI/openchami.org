# OpenCHAMI Weekly Digest

## Highlights
- **Enhanced DNS capabilities in coresmd:** Added support for the `rrl` plugin to improve DNS response rate limiting, plus proposed per-interface DNS entries for finer control. ([Issue #44](https://github.com/OpenCHAMI/coresmd/issues/44), [Issue #43](https://github.com/OpenCHAMI/coresmd/issues/43), [PR #45](https://github.com/OpenCHAMI/coresmd/pull/45))
- **Security upgrade in boot-service:** Introduced JWT support on the client side to enable secure, token-based authentication. ([Issue #4](https://github.com/OpenCHAMI/boot-service/issues/4))
- **Fabrica improvements:** Closed a key feature enabling API package generation for existing modules/apps, fixed case sensitivity for adding resources, and improved example documentation for easier onboarding. ([Issue #26](https://github.com/OpenCHAMI/fabrica/issues/26), [PR #32](https://github.com/OpenCHAMI/fabrica/pull/32), [PR #33](https://github.com/OpenCHAMI/fabrica/pull/33))
- **Metadata-service gets smarter:** Added new Profile resource with related operations, enhancing user metadata management capabilities. ([PR #6](https://github.com/OpenCHAMI/metadata-service/pull/6))
- **Remote-console update:** Refactored internals and exposed a new websocket interface, improving real-time connectivity and control. ([PR #15](https://github.com/OpenCHAMI/remote-console/pull/15))
- **Magellan quality boost:** Addressed multiple issues, enhanced log clarity, updated docs, and added regression tests to ensure stability and traceability. ([PR #126](https://github.com/OpenCHAMI/magellan/pull/126))
- **Community and governance:** Updated TSC membership details to keep the leadership info current and transparent. ([PR #40](https://github.com/OpenCHAMI/community/pull/40))
- **Documentation enhancements:** Added new "Install Slurm" guide and reorganized openchami.org content for better usability. ([PR #72](https://github.com/OpenCHAMI/openchami.org/pull/72), [PR #71](https://github.com/OpenCHAMI/openchami.org/pull/71))
- **Image-builder flexibility:** Added the ability to write repository configurations, giving users more control over builds. ([PR #56](https://github.com/OpenCHAMI/image-builder/pull/56))
- **Bug to fix:** Fabrica server experiences startup issues when using SQLite backend with Ent; fix pending. ([Issue #28](https://github.com/OpenCHAMI/fabrica/issues/28))
