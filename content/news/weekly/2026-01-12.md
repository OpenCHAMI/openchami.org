# OpenCHAMI Weekly Digest (Last 7 Days)

## Highlights
- **Storage Upgrade in tutorial-2025:** Proposal to replace MinIO with Versity Gateway for improved storage backend. [Issue #11](https://github.com/OpenCHAMI/tutorial-2025/issues/11)
- **Fixing Image Build on x86 Nodes:** Bug reported where image-build v0.1.2 fails on x86 architecture, impacting build consistency. [Issue #50](https://github.com/OpenCHAMI/image-builder/issues/50)
- **Community Standards Advancing:** A guide for coding standards and linters was completed to enhance code quality and consistency. [Issue #11 (closed)](https://github.com/OpenCHAMI/community/issues/11)
- **Better Repository Governance:** Proposal underway to add maintainers and CODEOWNERS files across repositories for clearer responsibility. [Issue #21](https://github.com/OpenCHAMI/community/issues/21)
- **Ochami CLI Modularization:** Refactoring commands and CLI library into separate packages to improve maintainability. [PR #66](https://github.com/OpenCHAMI/ochami/pull/66)
- **Announcing OpenCHAMI Developer Summit 2026:** Details added for the upcoming community event at UCL. [PR #65](https://github.com/OpenCHAMI/openchami.org/pull/65)
- **Fabrica Improvements:** Added kubebuilder-style versioning support and updated gitignore to include YAML files. [PR #27](https://github.com/OpenCHAMI/fabrica/pull/27), [PR #29](https://github.com/OpenCHAMI/fabrica/pull/29)
- **Enhanced CoreDNS Plugin:** New ‘header’ plugin being added to improve DNS middleware capabilities. [PR #42](https://github.com/OpenCHAMI/coresmd/pull/42)
- **More Flexible Image-Build Configs:** Introducing ability to add custom DNF arguments via config or command line for greater build customization. [PR #51](https://github.com/OpenCHAMI/image-builder/pull/51)
- **Community Architecture Decision Records (ADRs):** Added "Out Of Scope" ADR to clarify project boundaries. [PR #31](https://github.com/OpenCHAMI/community/pull/31)
