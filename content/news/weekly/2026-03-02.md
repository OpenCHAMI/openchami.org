# OpenCHAMI Weekly Digest: Recent Highlights (Feb 16–Feb 23, 2026)

## Highlights

- **Expanded Architecture Support**  
  Work is underway to add [aarch64 support](https://github.com/OpenCHAMI/opaal/issues/7) in container images, improving compatibility on ARM-based systems.  

- **Improved Health & Version Monitoring**  
  A proposal to add [health and version endpoints](https://github.com/OpenCHAMI/boot-service/issues/5) aims to enhance service observability in `boot-service`.  

- **More Flexible Hostname Management**  
  Introducing a [Configurable Hostname Policy](https://github.com/OpenCHAMI/coresmd/issues/46) in `coresmd` to offer greater control over DHCP plugin behavior.  

- **Fixes for Interface Filtering**  
  Addressing a bug where filtering network interfaces does not honor query parameters in [`ochami`](https://github.com/OpenCHAMI/ochami/issues/68), improving user experience when querying interfaces.  

- **Auth Enhancements Across Components**  
  Open PRs add [Tokensmith authentication and authorization](https://github.com/OpenCHAMI/fabrica/pull/34) as well as a [policy loader v2](https://github.com/OpenCHAMI/tokensmith/pull/7), strengthening security frameworks.  

- **FRU Tracker Updates**  
  Multiple merged improvements for `fru-tracker`, including migrating to [Ent ORM with SQLite](https://github.com/OpenCHAMI/fru-tracker/pull/7) and adding back collector code for better hardware tracking.  

- **Boot-Service Command Additions**  
  New commands are being introduced in `ochami`'s [boot-service module](https://github.com/OpenCHAMI/ochami/pull/70) to expand management capabilities.  

- **Database Connectivity**  
  `smd` PR to [use default DB port if unspecified](https://github.com/OpenCHAMI/smd/pull/85) aims to simplify setup and reduce configuration errors.  

- **Documentation Growth**  
  New docs added for installing Slurm on OpenCHAMI systems in [openchami.org](https://github.com/OpenCHAMI/openchami.org/pull/72), plus event agenda updates to keep community informed.  

- **Latest Releases**  
  - [`coresmd v0.5.7`](https://github.com/OpenCHAMI/coresmd/releases/tag/v0.5.7) brings latest improvements.  
  - `fru-tracker` released [`v0.2.0`](https://github.com/OpenCHAMI/fru-tracker/releases/tag/v0.2.0) and [`v0.2.1`](https://github.com/OpenCHAMI/fru-tracker/releases/tag/v0.2.1) for enhanced hardware asset tracking features.
