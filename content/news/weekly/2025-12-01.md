# OpenCHAMI Weekly Digest (Last 7 Days)

## Highlights
- **New Release Updates**  
  - BSS advanced with v1.32.2 release improving stability and features.  
  - Ochami launched v0.6.0 delivering new capabilities and bug fixes.  
  - Release repo updated to v0.1.2 enhancing container management reliability.

- **Bug Fixes & Stability Improvements**  
  - Resolved SELinux access issue in coresmd-coredns container to improve Corefile access reliability ([issue #39](https://github.com/OpenCHAMI/release/issues/39), [PR #40](https://github.com/OpenCHAMI/release/pull/40)).  
  - Fixed error when adding duplicate BMC entries in Ochami ([issue #55](https://github.com/OpenCHAMI/ochami/issues/55)).  

- **Security & Policy Enhancements**  
  - Ongoing effort to replace tokensmith’s internal policy engine with Casbin for improved policy management ([issue #5](https://github.com/OpenCHAMI/tokensmith/issues/5), [PR #6](https://github.com/OpenCHAMI/tokensmith/pull/6)).  
  - OAuth2 access tokens added to SMD requests in power-control for enhanced security ([PR #56](https://github.com/OpenCHAMI/power-control/pull/56)).  

- **New Features & Architecture Work**  
  - Ochami added support for new discover format and conversion scripts to streamline discovery ([PR #63](https://github.com/OpenCHAMI/ochami/pull/63)).  
  - Image-builder progressing on multi-architecture support and Go version port for broader platform compatibility ([PR #49](https://github.com/OpenCHAMI/image-builder/pull/49), [PR #44](https://github.com/OpenCHAMI/image-builder/pull/44)).  

- **Community & Process Improvements**  
  - Reopened tracker issue for maintainers and codeowner files across repos to clarify responsibilities ([issue #21](https://github.com/OpenCHAMI/community/issues/21)).  
  - Initiated guide for coding standards and linters to improve code quality and consistency ([issue #11](https://github.com/OpenCHAMI/community/issues/11)).  
  - Documented ADR 003 “Out Of Scope” to better define project boundaries ([PR #31](https://github.com/OpenCHAMI/community/pull/31)).
